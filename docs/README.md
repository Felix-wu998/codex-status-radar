# 文档目录

这个目录放 GitHub 仓库需要携带的项目材料。这里不是生产代码目录，而是产品、架构、研究和实施计划的整理版本。

## 分区

- `architecture/`：技术结构、数据流和仓库布局。
- `decisions/`：ADR 风格的稳定决策。
- `product/`：PRD、隐私草案和产品范围。
- `research/`：技术验证、仓库评估和市场研究。
- `superpowers/plans/`：可以按任务执行的实施计划。
